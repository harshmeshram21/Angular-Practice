Angular Lifecycle Hooks

Understand the purpose of each lifecycle hook in Angular.

  • Called before Angular initializes the component.
  • Used to initialize class properties and inject dependencies.
  • Does not have access to Angular bindings.
  • Use Case: Injecting services and initializing variables.

  • Runs once after the constructor.
  • Used for initialization logic, fetching data from APIs.
  • Use Case: Making API calls, setting default values.

  • Runs whenever Angular detects a change.
  • Used for manual change detection.
  • Use Case: Detecting changes in deeply nested objects.

  • Runs after content projection (ng-content) is initialized.
  • Executes only once after projected content is set.
  • Use Case: Accessing projected content inside a component.

  • Runs after Angular checks the projected content (ng-content).
  • Executes after every change detection cycle.
  • Use Case: Responding to changes in projected content.

  • Runs after child components and views are initialized.
  • Used to access ViewChild/ViewChildren elements.
  • Use Case: Getting DOM elements after they load.

  • Runs after every view update.
  • Used for handling dynamic content updates.
  • Use Case: Handling UI updates that depend on changes.

  • Runs before the component is destroyed.
  • Used for cleanup tasks, unsubscribing observables.
  • Use Case: Avoiding memory leaks by removing event listeners.
